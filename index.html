Temas

CLI

(Command Line Interface)

Comando: Instruccion que el usuario le da al sistema.

TIPOS

I. Mkdir -nombre-: Crea una nueva carpeta nombrada como -nombre- ("make directory").
II. Cd -nombre-: Entra a la carpeta -nombre-. ("change directory").
III. Cd: Sube un nivel en la estructura de carpetas.
IV. Cd.. : Va a la carpeta anterior/padre.
V. ls: Lista los archivos y carpetas de la ubicacion/carpeta actual.(list)
VI. Echo "texto": Muestra texto en consola o crea archivos con contenido.
VII cat -archivo.txt-: Mostrará el contenido del archivo -archivo.txt- en la terminal.
VIII.cat x.txt y.txt > z.txt creará un archivo Z.txt que contendrá los contenidos de X.txt e Y.txt.
IX. touch -archivo.txt-: crea un archivo vacío llamado archivo.txt.
X. Sort: Ordena los archivos alfabéticamente.

Git

I. git init: Inicializa un nuevo repositorio local.
II. git add archivo: Agrega el archivo archivo al área de staging.
III. git commit -m "mensaje": Guarda los cambios localmente con un mensaje.
IV. git config: Configura opciones de Git (usuario, email, etc.).
V. git remote add origin -URL-: Conecta el repo local al remoto (Generalmente GitHub) con la URL especificada.
VII. git push origin main: Sube los commits al repositorio remoto origin en la rama main.
VIII. git pull: Trae los cambios desde el repo remoto al local.

GitHub

I. Crear nuevo repo: Botón "New repository" en GitHub, completás datos y lo creás.
II. PATs (Personal Access Tokens): Token que reemplaza contraseña para autenticar en GitHub desde la terminal.
III. Pushear repo desde terminal: git init → git remote add → git add → git commit→git push.
IV. GitHub Pages: En Settings del repo, activás Pages con la rama main y carpeta `/root`.

HTML

Elementos y atributos

I. Elementos: Son las partes del documento HTML, pueden ser reconocibles porque poseen etiquetas como <p> <a> y <img>.
II. Atributos: Son palabras especiales utilizadas dentro de la etiqueta de apertura para controlar el comportamiento del elemento.
Otra definicion util seria; propiedades dentro de las etiquetas. Ejemplos: `src`, `href`, `alt`, `class`, `id`.

Los elementos de Texto

I. `<h1>` a `<h6>`: Son los encabezados (headers), los usamos como titulos o subtitulos. h1 es el más grande, h6 el más chico.
II. `<p>`: Son los parrafos del documento. Si tener que escribir mucho, lo que escribas va a ir en un párrafo.
III. `<ul>`: Lista desordenada.
IV. `<li>`: Elemento de lista.
V. `<a href="-texto-.html">`: Son enlaces a otras paginas.
Ejemplo: <a href="Pagina12.com.ar"> Apreta para ir a Pagina12 </a> ------> Apreta para ir a Pagina12

Las estructuras de los elementos

I. Bloque: Ocupa todo el ancho. Ej: `<div>`, `<p>`, `<h1>`.
II. En línea: Ocupa solo el ancho necesario. Ej: `<span>`, `<a>`, `<img>`.
III. Elementos vacíos: No tienen contenido interno, por lo que no pueden albergar texto u otros elementos HTML que no sean los especificados directamente dentro de la etiqueta de apertura.
Son facilmente reconocibles porque no requieren etiqueta de cierre, como por ejemplo </elemento>.
Algunos elementos vacios son: `<img>`, `<br>`.

Estructura semantica

I. `<div>`: Contenedor genérico. Es como una caja en la que podemos poner muchos elementos. Sirve mucho para organizarlos usando CSS
II. `<span>`: Contenedor en línea, a diferencia de div que es como un "cuadrado"
III. `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`: Contenedores o etiquetas que definen el significado del contenido que engloban.
Header tiene los encabezados, los footers los pies de paginas, etc
A la hora de organizar la estructura semantica del documento HTML, hay que imaginarlo como un documento normal, como una hoja que escribimos a mano, que tiene un titulo, subtitulos, fotos, formularios, lo que sea que necesitemos.

Head y recursos

I. `<title>`: Título del sitio en la pestaña. Es el texto que se muesstra arriba de todo en el navegador. El titulo de esta pagina por ejemplo es "Los ejercicios de las pruebas de TRINI"
II. `<link rel="stylesheet" href="archivo.css">`: Conecta a un CSS externo. En otras palabras, hace que la pagina se vea como diga archivo.css .
III. `<script src="archivo.js">: Vincula al documento a un Javascript externo. Bascamente sirve para que lo que escribimos en en archivo.js se ejecute en la pagina.
IV. `<img src="ruta/a/la/foto.jpg" alt="Descripción">: Pone la foto espicificada en el documento.
Si la foto esta en la misma carpeta que el index.html (la pagina),
entonces el atributo src de la etiqueta seria sin la ruta: asi`<img src="soyLaImagenQueQueresPoner.png" alt="Descripción">
Tener en cuenta que en lo que ponemos en src (source), tiene que coincidir con la ruta (camino para llegar a la ubicacion de la foto), el nombre y el tipo de archivo de la foto, si tenemos una foto png va ".png", si es un gif ".gif", si la foto se llama KUNDO ponemos KUNDO etc.

CSS

(Cascading Style Sheets) - Hojas de Estilo en Cascada.
Propiedades Generales

Cuando hablamos de propiedades, almenos en CSS, nos referimos a las caracteristicas de un elemento, ya sean de forma, espacio o color.

I. Width: Es el ancho del elemento.
II. Height: Es la altura.
III. Padding: Traducido del ingles como relleno, es el espacio interno entre el contenido de un elemento y su borde.
Si el padding es mas grande que la altura o el ancho del elemento, no se van a respetar las dimensiones originales y la maquina va a ensanchar o alargar el elemento todo lo necesario para que el padding se cumpla.
IV. Margin: O en español margen, es el espacio exterior minimo que hay entre el elemento y otros.
V. Border: Es el borde del elemento. Es por defecto invisible, si queremos que se vea tenemos que indicar la cantidad de pixeles de su grosor (px) y escribir "solid".

Flexbox

Flexbox es una herramienta CSS muy util para gestionar el espacio de la pagina y la ubicacion de los elementos, pero NO forma parte de las propiedades basicas de CSS
Tiene tres propiedades principales:

Justify-content

Ordena los elementos del contenedor en sentido horizontal, por ejemplo si tengo muchas fotos (elemento) en un div (contenedor) y quiero que esten a la izquierda, tendria que aplicar la propiedad que corresponda al div.
A la hora de usarlo sirve pensar en una linea horizontal, siendo su final su derecha y su principio su izquierda.
Acepta los siguientes "valores":
I. Flex-start: Alinea los elementos al lado izquierdo del contenedor.
II. Flex-end: Alinea elementos al lado derecho del contenedor.
III. Center: Alinea elementos en el centro del contenedor.
IV. Space-between: Pone espacios iguales entre los elementos, sin considerar los bordes. Por eso se siente mas comprimido
V. Space-around: Distribuye el espacio de igual forma entre elementos y los bordes del contenedor. Por eso se ve mas espaciado.

Align-items

Es como justify-content, pero en vez de ser horizontalmente, es verticalmente. Pensar en una linea que nace desde arriba y desciende y termina abajo ayuda.
Acepta los siguientes "valores":
I. Flex-start: Alinea elementos a la parte superior del contenedor.
II. Flex-end: Alinea elementos a la parte inferior del contenedor.
III. Center: Alinea elementos en el centro (verticalmente hablando) del contenedor.
IV. Baseline: Muestra elementos en la línea base del contenedor.
V. Stretch: los Elementos se estiran para ajustarse al contenedor.

Flex-direction

Define la direccion y el sentido en el que los elementos se distrubuyen
Acepta los siguientes "valores":
I. Row: Los elementos son colocados en la misma dirección del texto. II. Row-reverse: Los elementos son colocados en la dirección opuesta al texto. III. Column: Los Elementos se colocan de arriba hacia abajo. IV. Column-reverse: Los elementos se colocan de abajo hacia arriba.

Asignaciones

A la hora de darle estilo a los elementos tenemos tres formas:

I. Clase (.): Múltiples elementos diferentes no necesariamente del mismo tipo con características iguales o parecidas

SOY UN h1 TENGO LA MISMA CLASE QUE LA FOTO

II. Id(#): Para darle caracteristicas visuales unicas a un solo elemento.

YO SOY UNICO EN EL MUNDO (:

III. Etiqueta(<X>): Para hacer que todos los elementos de un determinado se vean iguales.
